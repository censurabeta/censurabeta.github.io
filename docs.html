Thanks for betatesting Censura!
Censura is super easy to use - this page should tell you everything you need to use Censura in your server.

Scroll down to get started!
The first step with any bot is to add it to your server. You can do so by clicking the button below: Add Censura Beta to Discord Censura asks for some sensitive permissions. Here is what they are used for:
  • Manage Messages is required since it is used to enforce the censor.
  • Ban Members and Kick Members are optional and used for AltProtect.
  • Manage Nicknames is optional and is used to enforce the censor in user nicknames.
While it is not strictly required, having a logging channel for Censura is highly recommended since it allows you to see what Censura is doing. It is also required to submit bug reports if something doesn't work properly.

The logging channel can be called whatever you like, and may be private. Ensure Censura has the following permissions in the channel:
  • View Channel for obvious reasons
  • Send Messages for obvious reasons
  • Embed Links since Censura sends logs in an embed
  • Attach Files since Censura may sometimes send images in logs
  • Use External Emotes (optional) so that the status emotes show up properly
Also make sure that your moderators have the Use Slash Commands permission somewhere so they can configure the bot. Once you've created the channel, use the /setloggingchannel command to set the logging channel:
/setloggingchannel channel
Set the Censura bot logging channel
Censura
By default, only the server owner or members with Administrator or Manage Server have access to commands. To allow other moderators to configure Censura, you can add the moderator role(s) with /modrole add:
/modrole add role
Add a role to the mod roles list
Censura
By default, moderators are exempt from the Censura filter. If you want, you can disable this with /modrole exempt (note that this does not affect owners/admins):
/modrole exempt exempt
Exempt users with mod roles from the filter
Censura
Now that everything is set up, you can add a word to the censor list. It's as easy as calling /censor:
/censor word
Add a word to the censor list
Censura
To remove a word from the list, call /uncensor:
/uncensor word
Remove a word from the censor list
Censura
If Censura is flagging something it shouldn't, use /whitelist to make sure Censura doesn't flag it. You can also use the whitelist button on a log message.
/whitelist word
Add a word to the whitelist
Censura
Once you've configured your lists, you're done! Censura will immediately begin filtering words in any channel it has Manage Messages permissions in.

Whitelisting allows Censura to ignore certain words that might normally get censored. Sometimes it may not work properly because whitelist checks occur after the language model is applied. To demonstrate, look at this example log message:
Censura Documentation
Found "word"
woord
wooooooooooooooooooooord
woord
<none>
0
5 ms
#general
Jump
symspellbigram
Censura Beta v2
Whitelist 'woord'
Report a bug
In this case, even though 'wooooooooooooooooooooord' was censored, the language model transformed it to 'woord'. In order to properly whitelist this case, you would want to add 'woord' to the whitelist. You can use the whitelist button under the log message, or call /whitelist word: woord.

Use this command to configure how Censura should act when it detects an alt account.
/altprotectaction action
AltProtect Beta: Set the action Censura takes on alt accounts
Censura
Arguments:
  • action: Action to take on alt accounts
    Ban alt accounts
    Kick alt accounts
Note: Censura will only try to perform what is set here when it has permissions. For example, if Censura is set to ban accounts but does not have the Ban Members permission, it will not try to kick the account. Make sure the permissions are set up correctly.

When enabled, any usernames or nicknames containing unprintable characters are converted to printable ones and set as a nickname.
/deobfuscatenames deobfuscate
Automatically convert unprintable usernames/nicknames to printable Latin characters
Censura
Arguments:
  • deobfuscate: Whether Censura should deobfuscate usernames/nicknames
    Deobfuscate names
    Leave names obfuscated
Note: Names that contain any characters that are not typable on a standard US English keyboard are converted. Keep in mind that accent marks (including dotless-i and umlauts), non-Latin characters (Arabic, Greek, Hindi, etc.) and ideographs (Chinese, Japanese, and Korean) will also be converted. Until Censura expands outside of English, this will have to remain, but it will be fixed as soon as it is feasible.
